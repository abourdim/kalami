Browser & Setup
Why doesn't the microphone work in Firefox or Safari? ▾
Kalami uses the Web Speech API for Arabic recognition, which is currently only supported in Google Chrome and Microsoft Edge. Firefox and Safari do not implement this API. Please open kalami-v2.1.html in Chrome or Edge.
✅ Supported: Chrome 33+, Edge 79+
❌ Not supported: Firefox, Safari, Opera
The mic button is greyed out — what's wrong? ▾
This means the browser did not detect a supported Speech Recognition API. This happens in unsupported browsers. Switch to Chrome or Edge and reload the page. If you're already on Chrome, make sure you're not in Incognito mode with mic permissions blocked.
I see a microphone permission error. How do I fix it? ▾
Click the 🔒 lock icon in Chrome's address bar → Site settings → set Microphone to Allow. Then reload the page. If the file is opened locally (file://), Chrome may restrict mic access — try serving it via a simple local server: python3 -m http.server 8080 then open localhost:8080/kalami-v2.1.html.
Does Kalami need an internet connection? ▾
Yes, for two things: (1) Chrome's Speech Recognition sends audio to Google's servers for processing — this requires internet. (2) The MyMemory translation API also requires a connection. The app itself is a single offline HTML file; only these two API calls need the internet.
💾 Translation Memory: repeated phrases are cached locally in-session and don't require network after first translation.

Speech Recognition
The Arabic recognition is inaccurate — how can I improve it? ▾
Try these steps: (1) Select the correct dialect matching how you speak (Egyptian, Maghrebi, Levantine, Iraqi, or MSA/Gulf). (2) Speak in a quiet environment with minimal background noise. (3) Speak at a natural pace — not too fast. (4) Use a good-quality microphone. (5) After recognition, edit the transcript directly before translating.
Can I use the Arabic keyboard instead of speaking? ▾
Yes! Click the ⌨️ Arabic Keyboard button (or press K) to open the full Arabic on-screen keyboard. Type your text, then click the golden ⚡ Translate key. You can also type directly in the transcript box using your OS Arabic keyboard layout.
What's the difference between Manual and Continuous modes? ▾
Manual: You control when to listen. Click to start, click to stop. Best for clear, deliberate speech. Continuous: The mic stays open indefinitely, translating each time you pause. Best for live conversations. Toggle between them with the switch in the controls bar.
What is Near-Live mode? ▾
Near-Live translates each phrase chunk as you speak, without waiting for you to stop. The browser fires "final" recognition results at natural speech pauses (~every few words). Near-Live hooks into these events and translates + speaks each one immediately. Expect ~2–3 second lag per phrase — close to a real interpreter, but not instantaneous.

Translation
Which translation service does Kalami use? ▾
Kalami uses the MyMemory API by Translated.net — a free translation service with no API key required. It supports Arabic to French and Arabic to English. The free tier allows 5,000 characters per day. For heavy usage, you can register at mymemory.translated.net for a higher quota.
Translation failed — what do I do? ▾
If you see a "Translation failed" error: (1) Check your internet connection. (2) You may have hit the MyMemory daily limit (5,000 chars/day on the free tier). (3) Try again after a few seconds. The Translation Memory cache means previously translated phrases still work instantly even when the API is unavailable.
What is Back-Translation? ▾
Back-translation verifies accuracy by translating the FR/EN result back to Arabic and showing it under each entry. If the back-translated Arabic is similar to what you said, the translation is likely accurate. Enable it with the ↩ Verify button. Note: this doubles API calls per translation.
What is Translation Memory? ▾
Translation Memory caches every phrase you've translated during the session. If you say the same phrase again, Kalami returns the cached result instantly — no API call, no network delay. You'll see a ⚡ From memory badge flash briefly. The cache is session-only and clears when you close the page.

Features
How do I export my translations? ▾
TXT: Click 📄 TXT in the controls bar to download a plain text file with all entries (Arabic + French + English + timestamps). PDF: Click 📑 PDF to open a formatted print page. Use your browser's Print → Save as PDF to save it. The PDF includes the Bismillah header and is bilingual-formatted.
How do I star/bookmark an entry? ▾
Every translation card has a ☆ star icon at the bottom. Click it to bookmark that entry — it turns gold (⭐) and the card gets a gold border. Click again to unstar. Starred entries survive search filters and are highlighted visually.
Can I search my translation history? ▾
Yes — there's a search box at the top of both the French and English panels. Type any keyword to filter entries in real time. Clear the search box to show all entries again. Search is independent per panel, so you can filter FR and EN differently.
What does the Speak language selector do? ▾
It controls which language is spoken aloud when Auto-play or Near-Live is active. 🇫🇷 FR = only French is spoken. 🇬🇧 EN = only English. Both = French first, then English. For Near-Live mode, choosing a single language (not Both) gives the fastest response since only one TTS utterance needs to complete before the next chunk.
The voice playback sounds robotic — can I improve it? ▾
Voice quality depends on which voices your operating system has installed. On Windows, install additional languages via Settings → Time & Language → Language and add French/English speech packs. On macOS, go to System Settings → Accessibility → Spoken Content → System Voice. More installed voices appear in Kalami's voice dropdown. Google voices (e.g. "Google Français") tend to sound the most natural.

Privacy & Data
Does Kalami store my voice or translations? ▾
No. Kalami is a single HTML file with no backend, no database, and no analytics. All data stays in your browser's memory for the current session only. When you close the tab, everything is gone. No cookies, no localStorage, no data sent to any Kalami server — because there is no Kalami server.
Does Google see my speech? ▾
Chrome's Web Speech API sends audio to Google's servers for recognition processing. This is handled by your browser, not by Kalami. If this is a concern, consider running the app without an active Google account signed in, or use a privacy-focused browser extension. The translated text (not audio) is sent to MyMemory for translation.
Is Kalami open source? ▾
Yes — Kalami is MIT licensed. The entire application is a single HTML file; you can read every line of code by viewing the page source (Ctrl+U in Chrome). Feel free to modify, redistribute, or build upon it.
